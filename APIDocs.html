Game API Documentation

Overview

This API provides a card game where users can register, buy cards, view profiles, list cards, and battle against a computer opponent. It is designed to enable quick game sessions where users can challenge the computer based on various card attributes.

The game is similar to the Top Trumps card games. As a player, you can register and get a starter deck of cards. Each card represents a hero from the Marvel universe and has 4 characteristics: strength, skill, size and popularity.

As an example, the cards will have characteristics like this...

Batman
Strength 22
Skills 17
Size 5.7
Popularity 9.8

To get new cards, you can fight with your cards against the computer. Each deck of cards is ordered, and you fight with the first card of your deck. You can check it and decide which characteristic is the most promising, then challenge the computer with this characteristic. If the computer's card has a higher value than yours in this characteristic, you lose your card. But if your card is better, the computer's card is yours, and you keep your card. In case of a draw, both players keep their cards.

If you are not satisfied with your current card(s) or if you have lost all your cards, you can buy one or more cards from the store. The new cards are added to the top of your deck.

This documentation will go through the API, explain each of the endpoints and even allow you to try them out....essentially to play the game.

API Details

The API can be installed by following the instructions here.

API Version: v1

Response Format: JSON

Base URL: http://localhost:4000
Note: This may be different if it is installed on a different machine. You can set the Base URL in the side bar on the left if you wish to try this out.

Authentication

This API does not currently require authentication. However, a playerID is necessary to access player-specific resources after registration. This ID is generated and returned when a new player registers.

Rate Limiting and Quotas

This API does not enforce any rate limits. Users can make unlimited requests; however, you may want to avoid excessive requests to maintain optimal server performance.

Known Issues

The following are current limitations and known issues with the API. We are actively working to resolve them in future updates:

  • Birthdate Validation: The birthdate parameter accepts only the first 10 characters, allowing some non-standard dates (e.g., "0024-99-99sasa"). Currently, there is no validation for legitimate dates.
  • Missing Email Validation: The email parameter does not have to be an existing email address. There is no validation of email addresses.
  • Lost PlayerID: If a player loses their playerID, it cannot be retrieved after registration. Users must keep this ID secure for future access.
  • Unlimited Card Purchases: Players can currently purchase an unlimited number of cards, and there is no restriction on the total number of cards a player can own.
  • No "Win" State: Due to the ability to continually buy cards and the fact that 0 cards does not initiate a "Win" state, the game can go on forever.
  • Duplicate and Inconsistent Cards: Cards are not unique, and their attributes may vary even if they have the same name. There is no guarantee of consistency between cards.
  • Battle Draws with Missing or Incorrect Field: Battles initiated with no field specified, or with an incorrect field, are permitted but always end in a draw.
  • Computer Interaction Limitations: The computer can only respond in a battle but cannot initiate games or select the attribute for play.
  • Missing Reset Feature: Currently, there is no way to reset or reinitialize a player’s set of cards once they are obtained.
  • No Limit on Total Cards in the Game: There is no overall cap on the number of cards that can exist in the game, allowing users to continually buy more cards.

Register

Register a new player.

Description

The Register endpoint is used to create a new player profile within the game. A successful registration will result in a playerID being returned with a starting set of cards.

Parameters

This table lists the Header/Body parameters that can be used with this endpoint.

Name Type Data Type Required Options Notes
Content-Type Header String ✗ application/json This is not strictly required for this endpoint. However, specifying Content-Type is considered good practice.
username Body String ✓ - The unique name of the player. Attempting to register an existing username will return Error Code: 1
birthdate Body String (YYYY-MM-DD) ✓ - Player’s birthdate in YYYY-MM-DD format. In this version of the game, only the first 10 characters are checked. Also, the dates are not checked to see that they are legitimate. For example, 0024-99-99sasa will be accepted. This is a known issue.
email Body String ✓ - This should be a valid email address, however this is not currently enforced.

Request

The example curl command below uses the current Base URL, which you can update via the input field in the sidebar. After copying the command, you can modify other parameters as needed.

Copied!
curl -X POST http://localhost:4000/register -H "Content-Type: application/json" -d '{"username": "john_doe", "birthdate": "1990-01-01", "email": "john@example.com"}'

Profile

Retrieve player profile information.

Description

The Profile endpoint returns the profile information for a player using their playerID. This allows users to access their registration details. The playerID is essential for authenticating the request, so must be kept after registration.

Parameters

This table lists the Header/Body parameters that can be used with this endpoint.

Name Type Data Type Required Options Notes
Content-Type Header String ✗ application/json This is not strictly required for this endpoint. However, specifying Content-Type is considered good practice.
playerID Header String (UUID) ✓ - A unique identifier assigned to each player upon registration. This cannot be retrieved after registration, so ensure that it is not lost. It follows the UUID format 8-4-4-4-12 (e.g., 6cda7300-ddd2-4f28-bf8e-9705fca6e933).

Request

The example curl command below uses the current Base URL, which you can update via the input field in the sidebar. After copying the command, you can modify other parameters as needed.

Copied!
curl -X GET http://localhost:4000/profile -H "Content-Type: application/json" -H "playerID: 6cda7300-ddd2-4f28-bf8e-9705fca6e933"

Response

This section shows an example of the JSON response returned by the endpoint. Below the JSON is a table which explains each of the fields returned. 


      {
        "username": "john_doe",
        "birthdate": "2024-01-01",
        "email": "john@example.com"
      }

Response Fields

Name Data Type Description
username String The unique username of the player, specified during registration.
birthdate String (YYYY-MM-DD) The birthdate of the player in the format YYYY-MM-DD, as provided at registration.
email String The email address associated with the player’s account.

Error Codes

Error Code Message Resolution
2 Unknown player ID: playerID Check the playerID that was used. If this cannot be found, register a different username OR restart the server and re-register your previous user. Note: All data will be deleted if you restart the server.

Next Card

Get the player's next card.

Description

The Next Card endpoint retrieves the player’s next available card, using their playerID for authentication. The cards are kept in a strict order so the next card will always be the same unless the Battle or the Buy Card endpoints are called. If no cards are available, the player will need to purchase additional cards using the Buy Card endpoint to continue playing.

Battle

Start a battle using a chosen attribute.

Description

The Battle endpoint allows a player to initiate a round against the computer using a specific attribute such as strength, skill, size, or popularity as the attribute to compare. The playerID also needs to be supplied. The player’s card is compared to the computer’s card, and the higher value wins. If no attribute is chosen, the battle ends in a draw.

Parameters

This table lists the Header/Body parameters that can be used with this endpoint.

Name Type Data Type Required Options Notes
Content-Type Header String ✗ application/json This is not strictly required for this endpoint. However, specifying Content-Type is considered good practice.
playerID Header String (UUID) ✓ - A unique identifier assigned to each player upon registration. This cannot be retrieved after registration, so ensure that it is not lost. It follows the UUID format 8-4-4-4-12 (e.g., 6cda7300-ddd2-4f28-bf8e-9705fca6e933).
field Body String ✗ strength, skill, size, popularity This parameter specifies the attribute used in the battle between the player’s card and the computer’s card. Only one of these fields can be chosen per battle. The chosen attribute will determine which card wins, based on its value. This is not required, but if it is not selected the round will always end in a draw. The value is also case sensitive. Anything other than the options provided will result in a draw. This is a known issue and will be handled in a future version.

Request

The example curl command below uses the current Base URL, which you can update via the input field in the sidebar. After copying the command, you can modify other parameters as needed.

Copied!
curl -X POST http://localhost:4000/battle -H "Content-Type: application/json" -H "playerID: 6cda7300-ddd2-4f28-bf8e-9705fca6e933" -d '{"field": "strength"}'

Response Fields

Name Data Type Description
outcome String The result of the battle, with possible values: win, loss, or draw. This indicates if the player's card won, lost, or tied with the computer's card.
card Object An object representing the player’s card used in the battle. It includes details about the hero and attributes that were used to determine the battle outcome.
card.id String (UUID) The unique identifier for the player’s card, following the UUID format.
card.name String The name of the hero on the player’s card.
card.strength Integer The strength attribute of the player’s card, ranging from 0 to 50.
card.skill Integer The skill attribute of the player’s card, ranging from 0 to 30.
card.size Float The size attribute of the player’s card, ranging from 5.0 to 11.0.
card.popularity Float The popularity attribute of the player’s card, ranging from 0.0 to 10.0.
opponentCard Object An object representing the computer’s card used in the battle. It includes details about the hero and attributes used in comparison with the player’s card.
opponentCard.id String (UUID) The unique identifier for the opponent’s card, following the UUID format.
opponentCard.name String The name of the hero on the opponent’s card.
opponentCard.strength Integer The strength attribute of the opponent’s card, ranging from 0 to 50.
opponentCard.skill Integer The skill attribute of the opponent’s card, ranging from 0 to 30.
opponentCard.size Float The size attribute of the opponent’s card, ranging from 5.0 to 11.0.
opponentCard.popularity Float The popularity attribute of the opponent’s card, ranging from 0.0 to 10.0.
